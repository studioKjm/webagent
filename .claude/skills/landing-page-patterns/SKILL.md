# Landing Page Patterns Skill

이 스킬은 **전환율이 검증된 랜딩페이지 구조와 패턴**을 제공합니다.

## 핵심 패턴: Hero-Problem-Solution-CTA 구조

```
┌─────────────────────────────────────────┐
│  HERO SECTION                           │
│  - 헤드라인 (가치 제안)                  │
│  - 서브헤드라인 (구체화)                 │
│  - CTA 버튼 (Above the Fold)            │
│  - 히어로 이미지/비디오                  │
│  - 높이: 100vh (전체 화면)               │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  SOCIAL PROOF                           │
│  - 고객 로고 (5-7개)                    │
│  - 사용자 수 / 통계                      │
│  - "10,000+ 개발자가 사용 중"            │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  PROBLEM SECTION                        │
│  - 타겟 오디언스의 고민 3가지            │
│  - "이런 문제 겪고 계신가요?"            │
│  - 공감 유도                             │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  SOLUTION SECTION                       │
│  - 제품이 문제를 해결하는 방법           │
│  - "우리는 이렇게 해결합니다"            │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  FEATURES                               │
│  - 주요 기능 3-6개                       │
│  - 각 기능: 아이콘 + 제목 + 설명         │
│  - 3 컬럼 그리드 (데스크톱)              │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  HOW IT WORKS (선택)                     │
│  - 사용 방법 3-4단계                     │
│  - "1. 가입 → 2. 설정 → 3. 실행"        │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  TESTIMONIALS                           │
│  - 고객 후기 2-3개                       │
│  - 사진 + 이름 + 직함 + 회사명           │
│  - 별점 또는 결과 수치                   │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  PRICING (선택)                          │
│  - 간단한 가격표 (1-3 플랜)              │
│  - 기능 비교표                           │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  FINAL CTA                              │
│  - 강력한 헤드라인으로 재유도            │
│  - "지금 바로 시작하세요"                │
│  - 위험 제거 문구 ("신용카드 불필요")    │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  FOOTER                                 │
│  - 링크 (회사 소개, 이용약관, 개인정보)  │
│  - 소셜 미디어 링크                      │
│  - 저작권 정보                           │
└─────────────────────────────────────────┘
```

## 전환율 최적화 원칙

### 1. Above the Fold 원칙
- **CTA는 스크롤 없이 보여야 함**
- 사용자가 페이지에 진입한 후 3초 안에:
  - 무엇을 제공하는지 이해
  - 누구를 위한 것인지 파악
  - 다음 행동(CTA) 명확히 인식

### 2. F-Pattern 읽기 패턴
- 사용자는 F자 형태로 콘텐츠를 읽음
- **중요한 정보는 왼쪽 정렬**
- 헤드라인, CTA 버튼은 왼쪽에 배치

### 3. 3초 룰
- 3초 안에 가치 제안 이해 가능해야 함
- 헤드라인은 짧고 명확하게 (60자 이내)

### 4. 단일 목표 원칙
- **한 페이지에 하나의 주요 CTA 목표**
- 여러 버튼이 있어도 모두 같은 행동 유도
- 예: "무료 체험", "지금 시작", "가입하기" → 모두 가입으로 이어짐

## CTA (Call-to-Action) 배치 전략

### 효과적인 CTA 위치

| 위치 | 평균 전환율 | 사용 상황 |
|-----|-----------|----------|
| **Hero (Above Fold)** | 35% | 즉시 전환 유도, 브랜드 인지도 높은 경우 |
| **Feature 섹션 후** | 25% | 가치 이해 후 전환, 제품 설명 필요한 경우 |
| **Testimonial 후** | 20% | 신뢰 구축 후 전환, 고가 제품 |
| **Footer** | 15% | 페이지 끝에서 재유도, 정보 소비형 사용자 |

### CTA 버튼 최적화

#### 색상
- **대비가 높은 색상 사용** (주변 배경과 차별화)
- 일반적으로 효과적인 색상:
  - 주황색 (#FF6B35) - 긴박감
  - 초록색 (#10B981) - 긍정, 진행
  - 파란색 (#3B82F6) - 신뢰, 안정

#### 크기
- **최소 48px 높이** (터치 친화적)
- 데스크톱: 56-64px 권장
- 충분한 패딩 (좌우 32px 이상)

#### 문구
| 일반적 | 최적화 | 전환율 증가 |
|--------|--------|------------|
| "시작하기" | "**무료로** 시작하기" | +25% |
| "가입" | "**30초 만에** 가입" | +18% |
| "더 알아보기" | "**3분 데모** 보기" | +32% |
| "다운로드" | "**지금 무료** 다운로드" | +21% |

**공식**: [행동] + [가치/시간/이점]

## 헤드라인 작성 공식

### 공식 1: 결과 중심
```
"[숫자]배 [결과] 달성"
"[시간] 만에 [원하는 결과]"
```

**예시**:
- "3배 빠른 웹사이트 빌드"
- "10분 만에 랜딩페이지 완성"
- "전환율 2배 향상"

### 공식 2: 문제 해결
```
"[고민/문제] 해결하는 [솔루션/제품]"
"더 이상 [부정적 경험] 하지 마세요"
```

**예시**:
- "복잡한 배포를 5분으로 단축하는 플랫폼"
- "더 이상 느린 웹사이트로 고객을 잃지 마세요"

### 공식 3: 대비 강조
```
"[A]는 [형용사], [B]는 [형용사]"
```

**예시**:
- "설정은 간단하게, 성능은 강력하게"
- "코드는 적게, 결과는 많이"

### 공식 4: 질문형 (공감 유도)
```
"[타겟]이 [문제] 겪고 계신가요?"
```

**예시**:
- "랜딩페이지 제작에 일주일씩 쓰시나요?"
- "배포할 때마다 불안하신가요?"

## 레이아웃 패턴

### 패턴 A: 단일 컬럼 (모바일 최적화)
```
[Hero - Full Width]
[Social Proof - Logos]
[Feature 1 - Full Width]
[Feature 2 - Full Width]
[Feature 3 - Full Width]
[Testimonial - Full Width]
[CTA - Full Width]
```

**사용 상황**:
- 모바일 우선 제품
- 콘텐츠가 많은 경우
- 스크롤 중심 경험

### 패턴 B: 번갈아 나타나는 2 컬럼
```
[Hero - 왼쪽 텍스트 / 오른쪽 이미지]
[Feature 1 - 왼쪽 이미지 / 오른쪽 텍스트]
[Feature 2 - 왼쪽 텍스트 / 오른쪽 이미지]
[Feature 3 - 왼쪽 이미지 / 오른쪽 텍스트]
[CTA - 중앙 정렬]
```

**사용 상황**:
- 제품 스크린샷이 많은 경우
- SaaS 제품
- 시각적 설명이 중요한 경우

### 패턴 C: 그리드 기반 (현대적)
```
[Hero - 중앙 정렬]
[Features - 3 컬럼 그리드]
[Testimonials - 2 컬럼 그리드]
[Pricing - 3 컬럼 카드]
[CTA - Full Width]
```

**사용 상황**:
- 현대적이고 깔끔한 느낌
- 기능이 많은 제품 (6개 이상)
- B2B SaaS

## A/B 테스트 우선순위

전환율에 가장 큰 영향을 미치는 요소 (우선순위 순):

1. **CTA 버튼 문구** (영향도: ★★★★★)
   - "시작하기" vs "무료로 시작하기"
   - "가입" vs "30초 만에 가입"

2. **헤드라인** (영향도: ★★★★★)
   - 결과 중심 vs 문제 해결
   - 짧은 헤드라인 vs 긴 헤드라인

3. **Hero 이미지 vs 비디오** (영향도: ★★★★☆)
   - 정적 이미지 vs 데모 비디오
   - 제품 스크린샷 vs 사람 사진

4. **CTA 버튼 색상** (영향도: ★★★☆☆)
   - 주황 vs 초록 vs 파랑

5. **폼 필드 개수** (영향도: ★★★★★)
   - 1개 (이메일만) vs 3개 (이름+이메일+전화)
   - **적을수록 전환율 높음**

## 사회적 증거 (Social Proof) 패턴

### 숫자 강조
```
"10,000+ 개발자가 선택한 플랫폼"
"월 1억 요청 처리"
"99.99% 업타임 보장"
"평균 응답 시간 50ms"
```

### 고객 로고
```html
<section class="social-proof">
  <p>신뢰할 수 있는 기업들이 사용합니다</p>
  <div class="logo-grid">
    <!-- 5-7개 고객 로고 -->
    <img src="company1.svg" alt="회사1">
    <img src="company2.svg" alt="회사2">
  </div>
</section>
```

### 고객 후기 구조
```
┌─────────────────────────────┐
│  "배포 시간이 1시간에서      │
│   30초로 단축됐어요."        │
│                             │
│  [프로필 사진]               │
│  김철수                      │
│  CTO, 스타트업 A             │
│  ⭐⭐⭐⭐⭐                   │
└─────────────────────────────┘
```

## 섹션별 체크리스트

### Hero Section
- [ ] 헤드라인 60자 이내
- [ ] 서브헤드라인으로 구체화
- [ ] CTA 버튼 Above the Fold
- [ ] 히어로 이미지/비디오 포함
- [ ] 100vh 높이

### Features Section
- [ ] 3-6개 기능 (너무 많으면 압도)
- [ ] 각 기능: 아이콘 + 제목 + 설명
- [ ] 그리드 레이아웃 (3 컬럼)
- [ ] 모바일에서 1 컬럼으로 변경

### CTA Section
- [ ] 버튼 크기 48px 이상
- [ ] 대비 색상 사용
- [ ] 행동 지향적 문구
- [ ] 위험 제거 문구 ("신용카드 불필요")

### Footer
- [ ] 중요 링크 (회사 소개, 이용약관)
- [ ] 소셜 미디어 링크
- [ ] 저작권 정보
- [ ] 연락처 정보

## 사용 예시

### 예시 1: SaaS 제품
```
타겟: 개발자
구조: Hero → Social Proof → Features (3 컬럼) → Code Example → Testimonials → Pricing → Final CTA
스타일: 다크모드, 코드 중심
```

### 예시 2: 마케팅 툴
```
타겟: 마케팅 팀
구조: Hero → Problem → Solution → Features (번갈아 2 컬럼) → Case Study → Testimonials → Final CTA
스타일: 밝은 색상, 대담한 타이포그래피
```

### 예시 3: 모바일 앱
```
타겟: 일반 소비자
구조: Hero (앱 스크린샷) → Social Proof → Features (단일 컬럼) → How It Works → App Store Badges → Final CTA
스타일: 미니멀, 모바일 우선
```
